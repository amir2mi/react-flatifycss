# Toggle switch

<p class="component-doc-intro">
  Toggle switch can be used instead of both checkbox and radio components, with
  a playful and reactive design.
</p>
<div class="funky-spacer" />

## Default

Specify the type of your toggle switch with `type` prop, it can be `checkbox` or `radio`.  
You can control the toggle switch checked state with `checked` prop, also get the checked state on change with `onChange` event like so:  
`onChange={(e, checked) => setChecked(checked)}`

## Uncontrolled

You can have an uncontrolled toggle switch as well, just do not use the `checked` prop and get the checked state with `onChange(event, isChecked)` argument.

  <Story id="inputs-toggle-switch--uncontrolled" />

## Radios

"Do not lose your mind, while you follow your heart." - UNKNOWN  
You have radios in form of toggle switches, give every radio a specific value like `value="mind"` and `value="heart"`, then check if the value is equal to `checked={checked === 'mind'}` or `checked={checked === 'heart'}`. Also set the selected switch `onChange` with the third parameter, like so:  
 `onChange={(e, checked, value) => setChecked(value)}`

<Canvas>
  <Story id="inputs-toggle-switch--radios" />
</Canvas>

## Uncontrolled radios

To have uncontrolled toggle switches that are radios technically, just do not set the `checked` prop. It is possible to set the default toggle to be checked with `defaultChecked` prop.
